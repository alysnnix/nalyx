# Nalyx

Personal NixOS configuration with public/private split for multi-host support: desktop, laptop, WSL, VM, and homelab.

## Quick Start

```bash
# Build system (desktop, laptop, wsl, vm, homelab)
up  # auto-detects private repo

# Or manually
sudo nixos-rebuild switch --flake .#desktop

# Validate without building
nix flake check --no-build

# Format code
nix fmt

# Enter devShell (installs pre-commit hooks)
nix develop
```

## Critical Rules

```
NEVER edit hardware-configuration.nix files manually
```

- They are auto-generated by NixOS
- Use `nixos-generate-config` to regenerate if needed

```
ALWAYS test configurations before applying
```

- Use `nix flake check --no-build` to validate
- Use `nixfmt` to format before commit

```
NEVER put personal data in the public repo
```

- Real emails, SSH keys, API tokens, and passwords belong in nalyx-private
- vars.nix in public repo must only contain safe defaults
- Secrets (SOPS files) live exclusively in the private repo

## Public / Private Architecture

```
~/nalyx/           (PUBLIC - this repo)
├── vars.nix       # Safe defaults (user@example.com, empty keys)
├── flake.nix      # Detects private repo, merges vars
├── private/       # Empty placeholder directory
└── ...

~/nalyx-private/   (PRIVATE - separate repo, optional)
├── vars-override.nix   # Real values merged over defaults
├── secrets/
│   └── secrets.yaml    # SOPS-encrypted passwords and tokens
├── scripts/            # Private shell scripts
└── .sops.yaml
```

### How hasPrivate Works

In `flake.nix`, the private input defaults to `path:./private` (empty placeholder). The detection logic:

```nix
hasPrivate = private != null && builtins.pathExists "${private}/vars-override.nix";
```

This means:
- **Without private repo**: `hasPrivate = false`, uses safe defaults, `initialPassword = "changeme"`, SOPS disabled
- **With private repo**: `hasPrivate = true`, merges vars-override.nix, enables SOPS, loads private scripts

The vars merging uses `nixpkgs.lib.recursiveUpdate`:

```nix
vars = nixpkgs.lib.recursiveUpdate (import ./vars.nix) (
  if hasPrivate then import "${private}/vars-override.nix" else { }
);
```

### Module Pattern with hasPrivate

Modules that need conditional private features receive `hasPrivate` and `private` via specialArgs:

```nix
{
  pkgs,
  vars,
  lib,
  config,
  hasPrivate ? false,
  private ? null,
  ...
}:
{
  # SOPS only when private repo is available
  sops = lib.mkIf hasPrivate {
    defaultSopsFile = "${private}/secrets/secrets.yaml";
    # ...
  };

  # Password: hashed from SOPS or fallback
  users.users.${vars.user.name} = {
    isNormalUser = true;
    # ...
  } // (
    if hasPrivate then {
      hashedPasswordFile = config.sops.secrets.password.path;
    } else {
      initialPassword = "changeme";
    }
  );
}
```

## Global Variables (vars.nix)

Public repo contains safe defaults:

```nix
{
  user.name = "aly";
  user.email = "user@example.com";
  user.publicKey = "";
  terminal = "kitty";
  editor = "nvim";
  desktop = "gnome";  # "hyprland", "gnome", or null
}
```

Private repo overrides with real values via `vars-override.nix`.

## Code Conventions

### Module Structure

```nix
# Standard module pattern
{ pkgs, vars, lib, hasPrivate ? false, private ? null, ... }:
{
  imports = [ ./submodule ];
  # configurations
}
```

### Conditional Imports

```nix
imports = [
  ./base
] ++ (lib.optional (vars.desktop == "hyprland") ./hyprland);
```

### Naming

- Directories: `kebab-case` or program name
- Main files: `default.nix`
- Specific configs: `descriptive-name.nix`

## Pre-commit Hooks

The project uses `git-hooks.nix` with:

- `nixfmt` - Nix code formatting
- `statix` - Nix linter
- `deadnix` - Dead code detection

Activated automatically with `nix develop`.

## Secrets (SOPS)

Secrets live in the private repo (`nalyx-private/secrets/secrets.yaml`).

```bash
# Edit secrets
cd ~/nalyx-private && sops secrets/secrets.yaml

# Secrets are decrypted at runtime
# Key: ~/.ssh/id_ed25519 (Age)
```
