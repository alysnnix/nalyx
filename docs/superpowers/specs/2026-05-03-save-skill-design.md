# `/save` Skill Design

**Status:** approved
**Date:** 2026-05-03
**Owner:** aly

## Goal

A user-invocable Claude Code skill, `/save`, that captures the current
conversation into the Obsidian-synced notes vault (`~/nalyx/.private/notes/`),
generates a human-curated summary, commits, and pushes — so conversations are
searchable in Obsidian and replicated across hosts.

## Non-goals

- Replacing the existing `notes` skill (which curates structural knowledge —
  `works/`, `personal/`, `meets/`).
- Editing the Claude Code session title (`/rename` cannot be invoked
  programmatically by a skill; the skill controls the saved filename only).
- Scrubbing secrets from transcripts. The user is responsible for reviewing
  before committing. Future hardening may add a redaction pass.
- Automated tests. The skill is small enough to validate manually.

## Vault layout

The skill writes under a new top-level folder `claude/conversations/`, parallel
to the existing `claude/works/`, `claude/personal/`, `claude/meets/`:

```
~/nalyx/.private/notes/claude/conversations/
├── index.md                                 # own index, not the global one
├── 2026-05-03/
│   └── design-skill-save/
│       ├── summary.md
│       ├── transcript.md                    # rendered, human-readable
│       └── transcript.jsonl                 # raw, faithful copy
└── 2026-05-04/
    └── ...
```

### Naming

- One folder per saved conversation: `YYYY-MM-DD/<slug>/`.
- `<slug>` is `kebab-case` of the title generated by Claude, max 60
  characters, ASCII only.
- Collisions on the same day get a numeric suffix: `<slug>-2`, `<slug>-3`.

### `summary.md` template

```markdown
---
type: conversation
title: "Design da skill /save"
session_id: 0803fc17-…
cwd: /home/aly/nalyx
host: wsl
date: 2026-05-03
duration_minutes: 42
tags: [claude-code, skill, automation]
---

# Design da skill /save

## Resumo
3-5 sentences: what was discussed, what problem was tackled, end state.

## Tópicos principais
- bullet
- bullet

## Decisões / artefatos produzidos
- files edited, commands run, PRs opened
- include important code snippets inline (fenced) when they aid understanding

## Próximos passos
- TODO if any

---
[transcript renderizado](transcript.md) · [transcript bruto](transcript.jsonl)
```

Frontmatter is required. `tags` are inferred by Claude from conversation
content (not derived from `cwd` or any heuristic).

### `transcript.md` rendering

The skill instructs Claude to read the raw `.jsonl` and render a readable
markdown version with this mapping:

| JSONL entry | Rendered |
|---|---|
| `type: file-history-snapshot` | dropped |
| `type: user`, string content | `## 👤 User\n\n<content>` |
| `type: user`, `tool_result` blocks | `> **Tool result** (<tool>):` + first ~20 lines + `... (truncated, N more lines)` if longer |
| `type: assistant`, `text` blocks | `## 🤖 Assistant\n\n<text>` |
| `type: assistant`, `tool_use` blocks | `> **Tool**: <name> <args summary>` |
| Embedded base64 images / `<system-reminder>` blocks | `_[image elided]_` / `_[system reminder]_` |

Header of `transcript.md`:

```
# <session-id-short> — <YYYY-MM-DD HH:MM> → <HH:MM>
**cwd**: /home/aly/nalyx | **branch**: main | **host**: <hostname>
```

`transcript.jsonl` is a verbatim copy of the source — no transformation. It
is the source of truth; `transcript.md` is for humans.

### `conversations/index.md`

A flat list, newest first, one line per saved conversation:

```markdown
# Conversations

- [Design da skill /save](2026-05-03/design-skill-save/summary.md) — designed
  the `/save` skill, simplified after pushback on over-engineering
- [Debug nginx ws](2026-05-03/debug-nginx-ws/summary.md) — fixed bare
  `/openclaw` ws upgrade in homelab nginx
```

Skill appends a new line each save; does not rebuild the file from scratch
(O(n²) avoided).

The global vault index (`~/nalyx/.private/notes/claude/index.md`) is **not**
touched — `conversations/index.md` is its own thing to keep the curated
global index from drowning in conversation entries.

## Components

A single artifact: the skill file itself.

```
home/features/cli/claude/skills/global/save/SKILL.md
```

No bash/python helper script, no Nix module beyond the skill mapping. The
skill orchestrates everything via the tools the LLM already has (`Bash`,
`Read`, `Write`, `Edit`).

This was a deliberate simplification — an earlier draft proposed two
Nix-managed shell scripts (one for JSONL parsing, one for git). They were
cut because the parsing fits in the LLM's existing capability and the git
flow is six lines of bash.

## Flow

```
user: /save
  ↓
skill instructs Claude to:
  ↓
  [1] resolve session paths
        - cwd        : `pwd` (always the active working directory)
        - encoded-cwd: cwd with every / replaced by -
                       (e.g. /home/aly/nalyx → -home-aly-nalyx)
        - jsonl_path : the most recently modified .jsonl under
                       ~/.claude/projects/<encoded-cwd>/ — that is the
                       active session
                       `ls -t ~/.claude/projects/<encoded-cwd>/*.jsonl | head -1`
        - session_id : basename of jsonl_path (used in summary frontmatter)
        - host       : `hostname`
  ↓
  [2] verify vault exists
        ls ~/nalyx/.private/notes/claude/ || abort with clear message
  ↓
  [3] read the JSONL
        Claude reads it via Read tool to inform title + summary + tags
        and to render transcript.md
  ↓
  [4] generate (in-context, no extra API call):
        - title    (human, e.g. "Design da skill /save")
        - slug     (kebab-case, ≤ 60 chars, ASCII)
        - summary  (4 sections per template above)
        - tags     (inferred from conversation)
        - rendered transcript.md
  ↓
  [5] create folder
        mkdir -p .../conversations/YYYY-MM-DD/<slug>/
        on collision: append -2, -3, …
  ↓
  [6] write artifacts
        - cp jsonl_path .../transcript.jsonl
        - Write transcript.md
        - Write summary.md
        - Edit conversations/index.md (prepend new line under "# Conversations")
  ↓
  [7] git
        cd ~/nalyx/.private/notes
        git pull --rebase origin main          # avoid host conflicts
        git add claude/conversations/YYYY-MM-DD/<slug>/ \
                claude/conversations/index.md
        git commit -m 'chore(conversations): save "<title>"'
        git push origin main
  ↓
  [8] report to user
        "Saved to conversations/YYYY-MM-DD/<slug>/, pushed to main."
```

## Failure handling

- **Vault missing** (`~/nalyx/.private/notes/` absent): skill prints
  `"Vault not found at ~/nalyx/.private/notes/. Run \`switch\` to clone it."`
  and aborts before creating any files.
- **JSONL not found** (no `.jsonl` under `~/.claude/projects/<encoded-cwd>/`):
  skill aborts with `"No active Claude Code session found for cwd <cwd>"`.
- **`git pull --rebase` fails** (true conflict): skill aborts after the
  files are written but before commit. User resolves manually. The skill
  does **not** force-push, does **not** stash, does **not** reset.
- **`git push` fails** (network, auth, rejected): files stay committed
  locally. Skill prints the git error and tells the user to retry the push.
- **Slug collision** (same date, same slug): suffix `-2`, `-3`. Skill checks
  by `ls` before mkdir.

## Nix integration

One edit:

`home/features/cli/claude/skills/files.nix` — add to `skillFiles`:

```nix
"save/SKILL.md" = ./global/save/SKILL.md;
```

That's it. Pre-commit hooks (`nixfmt`, `statix`, `deadnix`) handle
formatting. `nix flake check --no-build` verifies the build.

## Out of scope (future work)

- Secret scrubbing on the transcript before commit.
- Auto-pruning old conversations from the vault (could grow large).
- A reverse `/load <slug>` to seed a new session with prior context.
- Cross-host deduplication when two hosts both run `/save` simultaneously
  on different conversations (current `git pull --rebase` handles non-
  conflicting cases).
